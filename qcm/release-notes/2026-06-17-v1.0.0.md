### Security
- Hardened input handling in the admin form handlers: every `$_POST`/`$_GET`
  read is now unslashed and sanitized at the boundary with the correct sanitizer
  (`sanitize_text_field`/`sanitize_textarea_field`/`sanitize_email`/`esc_url_raw`/
  `absint`), clearing the `ValidatedSanitizedInput` warnings. Numeric strings
  (amounts) and multi-line descriptions are preserved; passwords are unslashed but
  intentionally not sanitized. (No active vulnerability existed — the model layer
  already sanitized before the database — but sanitizing at the boundary is correct.)
- Annotated the `NonceVerification` warnings: the flagged reads are read-only
  request vars (list/tab/navigation params) or `$_POST` reads in helpers whose
  calling handler already runs `check_admin_referer()`/`wp_verify_nonce()` — no
  CSRF gap existed; each is now marked with a justified `phpcs:ignore`.
- Annotated the custom-table DB-query warnings (`DirectQuery`, `NoCaching`,
  `SchemaChange`, `InterpolatedNotPrepared`, `UnescapedDBParameter`) in the
  repository classes and the activator — these are intentional direct queries on
  the plugin's own tables with trusted, constant table names.

### Fixed
- Plugin Check (errors): cleared all 301 shipped-code ERROR findings —
  - added `/* translators: */` comments to 114 placeholder i18n calls;
  - annotated trusted-constant table-name interpolations in prepared queries
    (`WordPress.DB.PreparedSQL.*` / `UnescapedDBParameter`) across the repository
    classes;
  - resolved every `EscapeOutput` finding (escaping the client form action with
    `esc_url()`, wrapping `$money()` output in `esc_html()`, casting numeric printf
    args to `(int)`, and annotating the already-escaped remainder);
  - scoped the attachment store's `move_uploaded_file()`/`rmdir()` ignores to the
    sniffs that flagged them; marked the standalone-document Google-Fonts `<link>`
    tags (PDFs + client pages) as intentional non-enqueued stylesheets;
  - added a `readme.txt`.
- The build zip ships only the runtime files (no `tests/`, `tools/`, `dist/`,
  `.DS_Store` or markdown) and now includes `readme.txt`, so the dev-only Plugin
  Check findings don't apply to the distributed plugin.

### Added
- Line-item tables show each line's unit suffix (/ project, / month, / each, / yr,
  / hour) next to the unit price, consistently across the quote detail, client quote
  view, quote PDF, contract detail, contract document/PDF and the renewal page.
- Quote → contract conversion now snapshots the full quote into the contract:
  a new `gqcm_contract_items` table copies every line item (with its unit and
  per-item financing) on accept, and the contract stores its own subtotal,
  discount, tax and a true cadence-aware **MRR**. The contract is now
  self-contained — editing or deleting the originating quote no longer changes a
  signed contract. The contract document, contract PDF and admin contract detail
  render the agreed line items + the same payment plan / schedule as the quote,
  from the contract's own data.
- Revenue rollup (TCV / MRR / ARR) now uses each contract's stored MRR (one-time
  fees excluded) instead of the crude `amount ÷ billing-cycle` estimate, which
  over-counted one-time fees. Manual/legacy contracts without a stored MRR keep
  the old estimate as a fallback.
- Renewals carry the snapshot forward: a successor contract created from a term
  whose price is unchanged inherits the predecessor's line items + financial
  breakdown + MRR, so renewed contracts stay faithful and keep counting correct
  revenue. A re-priced renewal does not copy a mismatched snapshot (it falls back
  to the estimate, to be adjusted).
- Renewals re-validate the original coupon: the contract now remembers the coupon
  code + amount it was created with, and a renewal only keeps that discount if the
  coupon is still active and unexpired. When the coupon is no longer valid the
  renewed term reverts to regular price — each line is re-priced to its current
  library (catalog) price, all promotional discount is dropped, and tax + MRR are
  recomputed. Custom lines (not in the library) keep their own price.
- Per-item renewal selection: the renewal page now has a "Renew" checkbox on each
  line — recurring (/mo, /yr) items default to renewing, one-time items default
  off. A one-time item chosen to renew can be billed Upfront or spread into a 3/6/12
  month payment plan. The totals and the Payment plan card update live as you select,
  and the renewed contract is built from exactly the chosen items. The Unit column
  shows each line's unit suffix (/ project, / month, / each …).
- "Renew at catalog price" toggle on the contract detail (Term & billing): when
  on, every renewal re-prices to current catalog prices and ignores any coupon —
  even one that's still valid. The preference carries forward to each successor.

### Removed
- The misleading per-contract "Send reminder" button on the contract detail (it
  actually ran the whole renewal sweep). Reminders fire automatically on the
  schedule; the manual sweep remains on the Dashboard and Settings pages.

### Changed
- Equivalent MRR is no longer shown on the client-facing quote or the quote PDF
  (it's an internal run-rate metric); it remains on the admin quote/contract
  detail pages.
- The contract's billing-cadence field is now labelled "Service period" (matching
  the quote) across the contract detail, contract document and new-contract form.
- The quote and contract detail pages now show the client's uploaded logo in the
  header and Client card, falling back to a colour-keyed initial avatar when none
  is set.
- The renewal/new-contract page matches the rest of the app: a hero header with
  the client avatar + "Renew C-…" title, uppercase section labels, and a line-item
  table with the same column order (Category · Service Group · Name · Unit · Qty ·
  Amount) and chips as the quote/contract detail pages, including the click-to-expand
  caret that reveals each line's markdown-rendered description. The preview reads from
  the contract's own snapshot (falling back to the source quote for legacy contracts).
