Smoke-test pass over the v0.7.181 admin surface flagged that Terminate was redundant with Suspend + Archive, and the user-facing Plan & billing page mis-read the state for DB-only trial workspaces.

### Changed

- **Reverted v0.7.181 Terminate workspace feature.** Suspend (reversible lock) and Archive (reversible hide) cover the involuntary-action use case; a separate lock-forever-no-credit path was more risk than value. Removed: \`POST /api/admin/workspaces/[id]/terminate\`, \`TerminatedBanner\`, \`TerminateWorkspaceButton\`, \`workspace-guard.ts\` helper, the assertWorkspaceWritable() calls in 6 write-route files, \`terminated_at\` reads in WorkspaceLite, the Danger Zone Card on \`/admin/workspaces/[id]\`. Schema columns (\`workspaces.terminated_at\`, \`termination_reason_*\`) + seeded \`billing.workspace_terminated\` template row stay in place as harmless dead artifacts.

### Added

- **Suspend → owner email notification.** PATCH \`/api/admin/workspaces/[id]\` with \`action='suspend'\` now fires a \`workspace.suspended\` email to the workspace owner (template seeded by migration 079). Until now the owner got zero signal that their workspace was locked. New \`sendWorkspaceSuspendedEmail()\` helper, \`email_sent\` lands in \`admin_audit.metadata\` for follow-up. Restore + Archive still silent on purpose.

### Fixed

- **Plan & billing page now branches on \`ws.plan\`, not \`stripe_subscription_id\`.** A workspace on a DB-only trial (paid plan, no Stripe sub yet because the user hasn't added a card) was falsely advertised as "You're on the Free plan — upgrade to unlock Pro features" while the upgrade page correctly showed "Current plan: pro · trialing". New \`isPaidPlan\` check (\`ws.plan !== 'free' && !isBeta\`) drives the branch; Stripe-dependent action buttons (Manage Portal, Cancel) gate separately on \`stripe_customer_id\` / \`stripe_subscription_id\`.
- **Price display reads \`plans.prices\` for the actual cadence.** Was hard-coded \`\${price_cents}/mo\` against the monthly column — a yearly Pro sub displayed as "Yearly · \$29/mo" (right cadence, wrong amount), and DB-only-trial workspaces with \`billing_interval=null\` got no price line at all. Now uses \`planPriceLabelForInterval(planRow, ws.billing_interval)\` which reads the \`prices\` jsonb (the same operator catalog managed at \`/admin/plans\`): yearly shows "\$278 / yr", trialing shows the monthly fallback "\$29 / mo".
