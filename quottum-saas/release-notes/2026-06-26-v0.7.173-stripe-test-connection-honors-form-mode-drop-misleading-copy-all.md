Two bugs in the v0.7.168 Stripe setup work, both honest UX cliffs.

## 1. Test connection now respects the form's mode toggle

**Before:** The Test connection button read the SAVED active mode from `admin_stripe_settings.mode` and tested THAT mode's key, ignoring the form's mode toggle. Result: flip to "Test" in the form (without saving) + click Test connection → got "LIVE mode confirmed".

**After:**
- API endpoint now accepts `body.mode` and tests that mode's saved key.
- Form passes `mode: form.mode` in the test request.
- New sanity check — if Stripe says the key's livemode contradicts the mode you asked to test (e.g. live key pasted into the test slot), the form shows a clear "Mode mismatch" error instead of cheerfully confirming the wrong thing.

## 2. "Copy all" on webhook events removed

The button I added in v0.7.168 promised something Stripe doesn't support — Stripe's event picker is search-as-you-type and collapses newlines to spaces, so pasting the joined list matched zero events. The per-row copy buttons (the actual right affordance) are unchanged.

Replaced "Copy all" with a small instructional note:
> Add one at a time — Stripe's event picker searches as you type and doesn't accept bulk paste. Use the copy button on each row, then paste into the search box and check the match.
