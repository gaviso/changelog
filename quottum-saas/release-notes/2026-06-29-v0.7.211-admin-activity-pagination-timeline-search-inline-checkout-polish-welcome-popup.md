### Admin console
- **Member activity** on the workspace-detail page is now paginated at **5/page** (shared `usePagination`/`TablePagination`; the global activity + user-detail pages are unchanged).
- **Workspace timeline** gets a **search box** — filters by event label, title, hint, and actor; the pager runs on the filtered set and resets to page 1 on query change.

### Inline checkout — Phase 1 polish (still behind `inline_checkout`, default OFF)
- **No more duplicate invoices:** the subscribe route cancels any existing `incomplete` subscription for the customer before creating a new one, so retries / double-clicks no longer pile up orphaned `open` invoices (Stripe issues one invoice per `default_incomplete` sub).
- **Stripe Link hidden:** `wallets.link:'never'` removes the "Save my information" email/phone signup + the Link banner on both the inline-checkout and update-card forms (Apple/Google Pay unaffected) for a clean card-only form.
- **Invoice status badges** now use the DS status tokens (`--success`/`--warning`/`--error` + `-light` tints) instead of hardcoded light `bg-*-50` classes, so they render correctly in dark mode.

### Subscription welcome popup
- A one-shot **"you're subscribed"** popup appears once after a workspace completes a paid subscription (inline Elements **or** returning Stripe Checkout — both land on the billing page). Paid vs trial copy by status. Reuses the existing modal + acknowledge pattern.
- Migration **090** adds `workspaces.first_subscription_welcome_seen_at` with a backfill that stamps existing active/trialing subscribers, so the popup only shows to **new** subscribers.

Built across five parallel/background agents (timeline search, member pagination, welcome popup, + the inline-checkout fixes), reviewed, composed type-check + lint clean.
