## Beta workspaces — real plan card on Settings → Billing
Beta-access workspaces used to see a bare *"You're on Beta access"* notice on `/dashboard/settings/billing`. Now they see a structured card matching the paid-workspace layout:

- Plan name with ✨ icon · **"Comped"** status chip · tester's role chip (Freelancer / Agency / Developer / etc.)
- ✓ Full app access *(with "X feature(s) reserved for paid plans" line when applicable — today the only excluded feature is white-label)*
- ✓ Storage cap shown (100 MB for the comp tier)
- ✓ Pointer to the **Submit feedback** sidebar button
- 📅 **Beta tester since** date pulled from the owner's `beta_testers.approved_at`
- "Ready to convert?" CTA with the conversion path (operator-assisted via `team@quottum.io`)
- Footer explains beta_access is a comp tier — no Stripe sub, that's why there's no "Manage billing" button

## Also fixed
- **Non-elevated members** on any plan now see the plan name on the billing page (previously they saw only an "ask your owner" message with no plan visibility).
