## Billing lifecycle emails — Phase 5 of 5 (Stripe rollout COMPLETE 🎉)

Customers now receive transactional emails for the four critical billing events. Card-expiring nudges are delegated to Stripe's built-in setting (operator one-time toggle).

| Stripe webhook | → | Email |
|---|---|---|
| `invoice.paid` (amount > $0) | → | **Receipt** with amount, plan, next-charge date, "Download invoice" CTA |
| `invoice.payment_failed` (amount > $0) | → | **Dunning** with "Update payment method" CTA to billing portal |
| `customer.subscription.trial_will_end` (3 days out) | → | **Trial-ending heads-up** with trial-end date + portal CTA |
| `customer.subscription.deleted` | → | **Cancellation confirmation** with downgrade notice + ask-for-feedback line |

All sends are wrapped in `.catch()` — an email failure can't crash the webhook handler. Zero-dollar invoices are suppressed (Stripe fires `$0 invoice.paid` at trial start; suppressing avoids a confusing "receipt for nothing").

## ⚠️ One-time operator setup

**Stripe Dashboard → Developers → Webhooks** → ensure the endpoint subscribes to (Phase 5 adds the last 2):
- `checkout.session.completed`
- `customer.subscription.created` / `updated` / `deleted` / `trial_will_end`
- **`invoice.paid`** *(NEW for Phase 5)*
- **`invoice.payment_failed`** *(NEW for Phase 5)*

**Stripe Dashboard → Settings → Emails** → enable **"Send emails about expiring cards"** *(NEW for Phase 5)*.
