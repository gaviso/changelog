Customer touchpoints (receipts, invoices, Customer Portal, hosted Checkout) were showing bare "Pro" / "Business" — no brand context. Stripe-side product + price names are now brand-prefixed.

| Where | Was | Now |
|---|---|---|
| Stripe product name | `Pro` | `Quottum Pro` |
| Stripe price nickname (monthly) | `Pro — monthly` | `Quottum Pro — monthly` |
| Stripe price nickname (yearly) | `Pro — yearly (20% off)` | `Quottum Pro — yearly (20% off)` |
| Internal `plan.name` (dashboard, billing card) | `Pro` | `Pro` (unchanged) |
| Same for Business | | |

Single change in [`src/app/api/admin/plans/[id]/sync-stripe/route.ts`](https://github.com/gaviso/quottum-saas/blob/v0.7.170/src/app/api/admin/plans/[id]/sync-stripe/route.ts) — derive `stripeName` once at the top of the handler, use everywhere a name goes to Stripe. Existing Stripe Product/Price IDs are preserved; only the `.name` and `.nickname` fields update on the next sync. Re-running the sync is idempotent.

## To apply the rename

Operator action required after this release deploys:

1. Open [`/admin/integrations`](https://app.quottum.io/admin/integrations) in operator mode
2. Click **Sync** on the **Pro** plan
3. Click **Sync** on the **Business** plan
4. Verify in Stripe Dashboard → Products that names are now "Quottum Pro" / "Quottum Business"

Repeat for both test and live modes if you want both renamed.
