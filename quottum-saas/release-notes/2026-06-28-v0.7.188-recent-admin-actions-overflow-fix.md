### Fixed

- **Long Stripe IDs no longer overflow the Recent admin actions card** on \`/admin/workspaces/[id]\`. Billing checkout audit entries render their full metadata inline (\`price_id: price_1Tn0q...\`, \`session_id: cs_test_b12Xdnk1Z...\`, \`customer_id: cus_Umj4...\`) which CSS treats as unbreakable words — they were breaking out of the card width. Added \`break-all\` to the action label paragraph so the IDs wrap mid-string inside the card.
